# The Demo Epoch is an explicit persisted scalar and `clock` is the sole business-time authority

The Demo Epoch — until now implicit, baked into seeded entity dates and never stored — becomes an explicit `demoEpoch` scalar persisted in the store snapshot, and every read or write of "now" flows through one seam: `clock.now()` returns the frozen instant `demoEpoch + offset`, and `clock.today()` returns its local-midnight start-of-day for the same-day comparisons ADR-0004 already mandates. Real wall-time survives in exactly two places — the initial `seedDemo()` and `resetDemo()`, which capture and re-anchor the epoch. This keeps ADR-0002's principle (dates are seed-relative so the demo never decays) but replaces its mechanism: a returning visitor's filters and write timestamps now read the frozen epoch instead of a live `new Date()` that had drifted past the seeded "upcoming" window and emptied the dashboards. `offset` is persisted, defaults to `0`, and is zeroed by reset; we ship frozen (no UI mutates it), so the planned date-travel feature only has to make `offset` settable — the seam does not change. We rejected a separate localStorage key for the epoch (a second source of truth that desyncs from the dates it anchors) and a clock that advances by real elapsed time within a session (it reintroduces the very decay the freeze exists to kill).

## Consequences

- `clock.now()` / `clock.today()` are the only business-time source: no component, hook, scope predicate, or store mutation calls `new Date()` / `Date.now()` for "now" — including write timestamps (`enrolledAt`, audit `timestamp`, `issuedAt`, `sentAt`) and the audit feed's relative time, so seeded and live records share one frozen timeline that never decays.
- Framework time stays real: React Query `staleTime` / `gcTime` and the api `_delay` are infrastructure, not demo time, and must not route through `clock`.
- `demoEpoch` is a required field in `isValidSnapshot`; a snapshot lacking it reseeds (ADR-0003), and this change bumps `STATE_KEY` to `v3`.
- Tests pin time by seeding a fixed epoch (`seedDemo(fixedDate)` in vitest; a Playwright `addInitScript` in e2e). With time frozen, the exact-date assertions ADR-0002 forbade become valid ("Session 3 of 8", "ends in 12 days") — and only genuinely time-driven test work-arounds are removed; async-resolution awaits are unrelated and stay.
