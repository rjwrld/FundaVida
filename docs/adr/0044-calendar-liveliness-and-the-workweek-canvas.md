# Every persona lands on a live week, and the calendar spends its pixels on the workweek

Revises the execution of ADR-0038 while keeping every invariant it inherited from ADR-0013: Sessions derive (ADR-0001), the surface rides the existing Courses scope with no new permission (ADR-0010/0013), and color stays semantic-only (the #239 Scarce Green decision). The revision is grounded in a diagnosis of why the shipped week agenda (#254) reads as broken. First, the **seed**: three of four personas have no in-progress Course — the teacher persona owns only completed/just-ended/upcoming cohorts, the student persona's live cohort starts thirteen days after the Demo Epoch, and the TCU trainee is assigned by an unconstrained faker roll that landed on an ended Course — so the landing week is empty by construction for every role but admin. Second, the **surface**: seven day-columns divide an app-shell content column into ~110px strips (guaranteed truncation, two permanently dead weekend columns, colliding day headers), the sidebar dumps unbounded all-time worklists ("176 sessions need marking" reads as abandonment, not operations), month mode's 2px/40%-opacity event marks render 24 marked days as none, and empty states dead-end. A portfolio demo's first paint is the product; both halves get fixed, and the seed half is load-bearing.

**The seed guarantees a live current week for all four personas.** The teacher persona gains an in-progress Course by reassigning the `teacherIndex` of an existing in-progress plan — not by adding a plan entry — so the Course count and the faker RNG sequence stay stable. The student persona enrolls in a currently-running cohort at their Sede (ADR-0011). The TCU persona is pinned to an in-progress Course deterministically after the faker selection, replacing the roll-of-the-dice assignment. The golden-path just-ended Course (ADR-0024) is untouched. Attendance seeding tops up past Sessions of in-progress Courses, leaving only the latest one to three unmarked per Course, so worklists read as a well-run school's remaining work — single digits, not 176. Stale snapshots reseed, never migrate (ADR-0003): STATE_KEY bumps, the constant is finally **exported** from `persistence.ts`, and e2e's `pinDemoEpoch` imports it — closing the silent-drift class for good.

**Agenda buckets are active-Courses-only.** `needsMarking` and the admin pulse count unmarked Sessions in in-progress Courses only; the ended-Course backlog is close-readiness's business on the Courses page (it already blocks closing there). The teacher's worklist groups by Course — one row, a count, a deep link to the oldest unmarked Session's mark page — never a row-per-session wall.

**The canvas is a workweek.** Monday–Friday columns always; a weekend column renders only when it actually contains a Session (users can author weekend meeting days; the seed never does). Cards drop the "({mon yyyy})" cohort suffix — a calendar already communicates _when_; the full name lives in the accessible name and tooltip — and show a two-line clamped title over `{Sede} · Session {n}/{total}`. Status leaves the Badge: a student's own verdict becomes a 3px edge rail plus one word in the verdict hue; a past-unmarked Session for teacher/admin becomes the page's one Figure-Green action, a "Mark attendance →" row. Time gets depth without new color: past cards mute, today's column and cards carry the tint, the future sits quiet.

**Month is a navigator, not a second calendar.** Event marks scale with the day's Session count so the term's shape is legible at arm's length, and tapping a day swaps to Week view positioned on that week. The day-detail panel below the grid is removed — the week canvas is that panel, done properly.

**Quiet states point somewhere; done states celebrate.** An empty week names the nearest Session in each direction and offers a jump to its week. An empty worklist or zero pulse reads as health — a quiet check, not a hole. Every such state is reachable by real interaction with seeded data: mark the last pending Session and the caught-up state appears because the data changed.

**Responsive is a ladder with one render path.** At `lg+`, the sidebar-plus-canvas split. Between `md` and `lg`, the five columns take the full width and the sidebar compresses to a one-row action banner above the canvas — the role's single most actionable fact — with the full buckets below. Under `md`, the _same_ five columns become CSS scroll-snap panels (~three-quarters viewport width, opened centered on today, the next day peeking as the scroll affordance). A week-strip day pager was considered and rejected: it introduces selected-day state and a second render structure for the same data — the DataTable dual-render lesson (#157). The calendar is the hero at every width; today's shipped mobile buries it under a full-height sidebar.

**The dashboard agenda rides along.** `AgendaSlice` adopts the same row language — de-suffixed names, tabular ordinals, one deep-linked hero fact, at most four rows, an "Open Calendar →" tail — so the first surface a reviewer sees matches the calendar they'll open next.

## Consequences

- Seed: in-progress plan reassignment to the teacher persona, student enrollment in a running cohort, deterministic TCU pin, attendance top-up. STATE_KEY v10 → v11, exported; e2e imports it; e2e anchors re-derive from `seedDemo()` since the RNG stream shifts.
- This package ships **before** milestone 5; #249's planned bump becomes v11 → v12 and its STATE_KEY-export criterion is satisfied here.
- Lib: `needsMarking`/pulse gain the active-Course filter; a nearest-Session derivation backs the empty week; a card-name variant drops the cohort suffix (`shortCourseName` is untouched — other surfaces still want the Sede-stripped name with the period).
- Components: `WeekCanvas` goes 5-plus-conditional-weekend with snap-panel styling under `md`; `SessionCard` gains rail/action anatomy; `AgendaSidebar` gains the grouped worklist and a banner variant; `CalendarWidget` gains density marks and a navigate-on-select callback; the month day-panel and `useDaySessions`'s panel role are removed; `AgendaSlice` adopts the row language.
- i18n: new EN+ES keys throughout, including a proper `calendar.sidebar.upcomingEmpty` (the Upcoming bucket currently misuses `calendar.emptyDay`).
- Tests: unit for active-only buckets, nearest-Session, weekend-conditional columns; component for card anatomy and grouped worklist; e2e asserts each role's landing week is non-empty, the mark flow shrinks the worklist to the caught-up state, month-tap navigation, and a mobile-viewport pass — locators via `getByRole` (#157). Axe coverage rides #208's broadened pass.
